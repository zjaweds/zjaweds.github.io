Errors in Competitive Coding

In Competitive coding , participants write source code to solve a well defined problem. The fundamental purpose of these codes is to solve specific problems optimally. So to prepare the participants for the purpose, online platforms provide well defined problems with certain time and memory constraints and participants need to write source code that passes all the testcases and time and memory used are calculated for each of the test cases and then compared with the allowed limits. These values determine whether a program is optimal or not.

          Because of these specified limits on time and memory, some other errors might appear into the scene, besides the errors in a programming language. In this article, we are trying to elaborate general errors that are commonly faced by competitive programmers.

  • Run TimeError:

      This error has nothing to do with your code; it often occurs due to lost connection between your system and the server of the platform where you have submitted the source code. Check your connectivity and you are good to go if the problem is at your end otherwise it will be taken care of by the host, which may be the rarest of the rare case.

    •                                                       

  • TLE: Time Limit Exceeded

      It occurs when your code does not execute all the test cases in the time constraint specified in the problem. It needs logical thinking and analysis of the algorithm being used. It seeks optimization of the algorithm. Time complexity has a crucial role in programming. Reduction of the time is the most important challenge and it is one of the core purposes of competitive coding.

    •                                                       

  • SIGSEGV: Signal Segmentation Violation

      It is a signal from the process to the OS. This error occurs when your program tries to access read only memory or the part of the memory which you are not allowed to use. Accessing the variable undeclared in the current scope

      1. Use of uninitialised pointers
      2. Stackoverflow due to non terminating recursion.
      3. Dereferencing Null Pointer
      4. Manipulation of string literals

      void rec_func(){
      int q = 5;
      rec_func();
      }
      int main(){
      rec_func();
      }

      This small code keeps on calling rec_func() without any termination, so it will cause stack overflow and finally it will be aborted by the operating system.

    •                                                       

  • SIGABRT: Signal Abort

      It is a signal sent by the process to the operating system in case of some serious issue which may lead to fatality of the OS. It calls abort() and forcibly terminates the execution of the program. It occurs in competitive coding due to problems in memory allocations due to internal error or due to violation of some constraints which may cause damage to the system.

      // C++ program to illustrate
      // the SIGABRT error
      int a = 100000000000;

      Assigning a value which occupies memory beyond the capacity of an int variable in c++.

    •                                                       

  • SIGFPE: Signal Floating Point Exception

      It is caused by all arithmetic errors that may cause fatality like division of a floating point number by zero, modulo operation by zero and integer overflow. If a program stores integer data in a location which is then used in a floating-point operation, this often causes an “invalid operation” exception, because the processor cannot recognize the data as a floating-point number.

      int a = 5;
      // Division by Zero
      cout << a / 0;

      Overflow: It is caused when a value which is out of range for the variable at the left hand side of the assignment operator is assigned as a result of arithmetic operation.

      int x=10;
      int y;
      y = pow(x,11);

      Above statement is trying to assign 100000000000 to an int variable y which is beyond its capacity. It doesn’t always lead to that error but it will give an unusual output.

    •                                                       

  • NZEC: Non Zero Exit Code

      In C language , it is caused by a program not having a return 0; statement in the main() function, but in C++ or Java, one of the following may be the reason.

      1. Infinite Recursion or if you run out of stack memory.
      2. Negative array index is accessed.
      3. ArrayIndexOutOfBounds Exception.
      4. StringIndexOutOfBounds Exception.
